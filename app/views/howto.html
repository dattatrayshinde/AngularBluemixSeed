Delivery Pipeline

The pre-configured pipeline executes each time you push to the Git repo. The pipeline automates your builds and deployments to IBM Bluemix and consists of three stages:
Build, Run Unit Tests, Active Deploy

If a job or a stage fails, the remaining stages will cancel.

  • Build

    The first stage is the build stage, where the pipeline will build and optimize our files for production. It mainly executes the the grunt command "grunt build" which will:

    • download our dependencies with bower & npm,
    • lint our code,
    • concatenate and minify our scripts and styles to save on those network requests,
    • optimize images if we were using any,
    • compile the output of any preprocessors weâ€™re using, and
    • generally make our application really lean.

  • Run Unit Tests

    The second stage is testing our code by executing the grunt command "grunt test:dist" which runs our Jasmine unit-tests in Karma. It will test our app's functionality with each deploy, making it easier to catch bugs as our app gets bigger and when more developers joins the team

  • Active Deploy

    The third and final stage is where the Active Deploy magic happens. It will deploy the application to your destination, without downtime. The Active Deploy pipeline stage information is fully described here.

The pipeline is also configured with Slack, which will notify you of every successful or failed job and stage. By sending messages to the slack-channel of your choice.
